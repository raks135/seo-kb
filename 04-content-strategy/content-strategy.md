## TL;DR
Effective content strategy matches content type to search intent, builds topical depth through pillar + cluster "hubs," and demonstrates E-E-A-T. Google rewards people-first content that shows first-hand experience; the "helpful content" system (integrated into core ranking in March 2024) demotes content created primarily for search engines.

## Core explanation
Content strategy answers *what to publish, for whom, and how it fits together*. The unit is the **query + intent**, not the keyword. Group related content into **topic clusters**: a comprehensive pillar page linked to narrower cluster pages, signaling topical authority to search engines.

## Mechanics / how-to
1. Map intents (informational, navigational, transactional, commercial — see 00-foundations/search-intent) to formats (guide, compare, product, hub).
2. Build clusters: one pillar + several supporting articles, interlinked.
3. Demonstrate E-E-A-T (S9): author bios, citations, first-hand evidence, especially for YMYL.
4. Align with "helpful content" principles (S25/S18): written for people, depth over breadth of thin pages.
5. Use entities (see 04-content-strategy/entity-seo): name real things, organizations, people so the Knowledge Graph can connect them.

## Worked example / code
Cluster outline for "espresso machines":
- Pillar: /espresso-machines/ (buying guide, types, how they work)
- Clusters: /espresso-machines/breville-review, /espresso-machines/grind-size, /espresso-machines/descale
- Internal links: each cluster → pillar; pillar → each cluster.

## Assumptions & limitations
- "Topical authority" is inferred from coverage + internal linking + external recognition; Google doesn't publish an "authority" score.
- Helpful-content classification is site-level-leaning but operates on signals; Google has not disclosed exact weights (S18, S25).
- E-E-A-T is most heavily weighted for YMYL; less decisive for trivial topics (S9).

## Empirical evidence
- Google's people-first content guidance and the March 2024 integration into core systems (Search Engine Land library, S25) are documented.
- Marie Haynes' case analyses (S27) show sites with thin/aggregated content disproportionately affected by helpful-content signals.
- Correlation studies link topical depth to ranking stability, but causation is inferred.

## Conflicting views
- **Pillar/cluster is mandatory.** It's a strong organizing model, not a ranking requirement; Google indexes individual pages by merit.
- **"More content = more traffic."** Google warns against scaling unhelpful content; thin mass-publishing can hurt (S1, S18).

## Common mistakes
- Publishing for search engines, not users (helpful-content risk).
- Orphan pages (no internal links).
- Ignoring intent mismatch (e.g., a product page ranking for "how-to" queries).
- Weak E-E-A-T on YMYL topics.
- Duplicating content across many near-identical pages.
